# 한글파일 TXT 변환 기능 기획서

## 1. 기획 개요

한글 문서(`.hwp`, `.hwpx`)에 포함된 텍스트를 추출하여 `txt` 파일로 변환하는 기능을 기획한다.  
사용자는 복잡한 편집 기능보다 문서 안의 텍스트를 빠르게 추출하고, 메모장이나 다른 시스템에서 재활용할 수 있기를 원한다.

이 기능은 다음과 같은 상황에서 유용하다.

- 문서 내용을 빠르게 확인해야 할 때
- 다른 시스템에 텍스트를 붙여넣어야 할 때
- 원본 문서의 서식은 필요 없고 내용만 필요할 때
- 대량 문서를 일괄 텍스트화해야 할 때

## 2. 목표

- 한글 문서를 손쉽게 `txt` 파일로 변환한다.
- 문서 본문의 텍스트를 최대한 정확하게 추출한다.
- 사용자가 별도 편집 없이 바로 저장하거나 다운로드할 수 있게 한다.
- 단일 파일 변환과 향후 다중 파일 변환 확장 가능성을 고려한다.

## 3. 핵심 사용자

- 한글 문서를 텍스트 데이터로 보관하려는 일반 사용자
- 문서 내용을 검색용 데이터로 바꾸려는 사무직 사용자
- 한글 문서 내용을 다른 프로그램에 입력하려는 운영 담당자
- 대량 문서 텍스트 추출이 필요한 관리자 및 실무자

## 4. 핵심 문제

한글 파일은 국내 환경에서는 자주 사용되지만, 다른 시스템이나 간단한 텍스트 처리 환경에서는 바로 활용하기 어렵다.  
특히 메모장, 스크립트, 데이터 분석, 검색 인덱싱 등에서는 순수 텍스트 형식이 더 적합하다.

현재 사용자는 다음과 같은 불편을 겪는다.

- 한글 프로그램이 없으면 내용을 다루기 어렵다.
- 복사/붙여넣기 과정에서 서식이 섞이거나 누락된다.
- 여러 파일을 반복적으로 열어 저장해야 해 번거롭다.

## 5. 해결 방안

사용자가 한글 파일을 업로드하면 내부적으로 텍스트를 추출하고, 결과를 `txt` 파일로 저장 또는 다운로드할 수 있게 한다.

기본 처리 흐름은 다음과 같다.

1. 사용자가 `.hwp` 또는 `.hwpx` 파일을 선택한다.
2. 시스템이 파일 형식을 판별한다.
3. 문서 내 텍스트를 추출한다.
4. 줄바꿈과 공백을 정리한다.
5. `.txt` 파일로 저장하거나 다운로드한다.

## 6. 주요 기능

### 6.1 파일 업로드

- `.hwp`, `.hwpx` 파일 업로드 지원
- 드래그 앤 드롭 또는 파일 선택 방식 제공
- 지원하지 않는 형식 업로드 시 안내 메시지 표시

### 6.2 텍스트 추출

- 문서 본문 텍스트 추출
- 문단 단위 줄바꿈 유지
- 불필요한 제어문자 제거
- 가능한 범위 내에서 표, 머리말, 꼬리말 텍스트도 추출 고려

### 6.3 TXT 변환 및 저장

- UTF-8 인코딩 기반 `txt` 파일 생성
- 원본 파일명 기준으로 `.txt` 확장자 저장
- 변환 결과 미리보기 제공 가능

### 6.4 예외 처리

- 암호화된 파일
- 손상된 파일
- 텍스트가 거의 없는 문서
- 이미지 중심 문서

## 7. 사용자 시나리오

### 시나리오 1: 단일 파일 변환

사용자는 한글 문서 1개를 업로드한다.  
시스템은 텍스트를 추출해 `txt` 파일로 변환하고 다운로드 버튼을 제공한다.

### 시나리오 2: 문서 내용 확인 후 저장

사용자는 업로드 후 추출된 내용을 화면에서 먼저 확인한다.  
원하는 경우 `txt` 파일로 저장한다.

### 시나리오 3: 변환 실패 안내

손상되었거나 지원되지 않는 문서일 경우, 시스템은 실패 사유와 함께 다시 시도할 수 있는 안내를 제공한다.

- 파일 업로드 영역
- 지원 형식 안내
- 변환 버튼

### 결과 화면

- 원본 파일명
- 변환 상태
- 추출 텍스트 미리보기
- `txt` 다운로드 버튼
- 재업로드 버튼

## 9. 기능 요구사항

- 사용자는 한글 파일을 업로드할 수 있어야 한다.
- 시스템은 업로드된 파일 형식을 검증해야 한다.
- 시스템은 문서에서 텍스트를 추출해야 한다.
- 시스템은 결과를 UTF-8 `txt` 파일로 저장할 수 있어야 한다.
- 변환 실패 시 사용자에게 원인을 안내해야 한다.

## 10. 비기능 요구사항

- 작은 문서는 빠르게 변환되어야 한다.
- 특수문자 및 한글 인코딩이 깨지지 않아야 한다.
- 업로드 파일은 안전하게 처리되어야 한다.
- 향후 대량 변환 기능 추가가 가능해야 한다.

## 11. 기술 검토 포인트

- `.hwp`와 `.hwpx`는 구조가 다르므로 분리 대응이 필요하다.
- `.hwpx`는 XML 기반이라 비교적 처리 가능성이 높다.
- `.hwp`는 전용 포맷 처리 라이브러리 또는 외부 도구 검토가 필요하다.
- 변환 정확도는 문서 구조, 표, 특수 객체 포함 여부에 따라 달라질 수 있다.

## 12. 우선순위

### 1차 개발

- 단일 파일 업로드
- `.hwpx` 우선 지원
- 텍스트 추출
- `txt` 다운로드
- 기본 예외 처리

### 2차 개발

- `.hwp` 대응 강화
- 결과 미리보기 개선
- 다중 파일 일괄 변환
- 압축 다운로드

## 13. 성공 기준

- 사용자가 한글 문서를 문제없이 업로드할 수 있다.
- 추출된 텍스트가 읽을 수 있는 수준으로 정확하다.
- 변환 결과를 `txt` 파일로 저장할 수 있다.
- 실패 케이스에서도 원인을 이해할 수 있다.

## 14. 참고 메모
