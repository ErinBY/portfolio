# 키오스크 주문 웹페이지 기획서

## 1. 프로젝트 개요

### 1-1. 프로젝트명
- 키오스크 주문 웹페이지

### 1-2. 프로젝트 목적
- 사용자가 매장 내 키오스크 화면에서 쉽고 빠르게 메뉴를 탐색하고 주문할 수 있는 웹페이지를 제작한다.
- 직관적인 UI와 최소한의 동선으로 주문 시간을 줄이고 사용 편의성을 높인다.
- JavaScript 기반으로 구현하여 화면 전환, 장바구니, 옵션 선택 등 상호작용을 부드럽게 제공한다.

### 1-3. 핵심 방향
- 언어: JavaScript
- 플랫폼: 웹 브라우저 기반 키오스크 페이지
- 디자인 방향: 깔끔하고 밝은 화이트 베이스
- 사용자 경험 목표: 누구나 쉽게 사용할 수 있는 단순하고 명확한 구조

---

## 2. 타겟 사용자

- 매장을 방문해 직접 주문하는 일반 고객
- 디지털 기기 사용이 익숙하지 않은 사용자
- 빠른 주문을 원하는 사용자

### 사용자 고려사항
- 버튼 크기를 크게 설계해 터치 조작이 편해야 한다.
- 한 화면에서 너무 많은 정보를 보여주지 않아야 한다.
- 현재 진행 상태를 쉽게 이해할 수 있어야 한다.
- 결제 전 주문 내역을 명확하게 재확인할 수 있어야 한다.

---

## 3. 서비스 목표

- 메뉴 탐색부터 결제 직전 확인까지의 흐름을 단순화한다.
- 메뉴 사진, 이름, 가격을 직관적으로 보여준다.
- 장바구니 확인과 수정이 쉬워야 한다.
- 사용자 실수를 줄이기 위해 선택 상태와 합계 금액을 명확히 표시한다.

---

## 4. 주요 기능

### 4-1. 메인 화면
- 브랜드 또는 매장명 노출
- 주문 시작 버튼 배치
- 추천 메뉴 또는 인기 메뉴 간단 노출

### 4-2. 카테고리 선택
- 예: 버거, 음료, 세트, 사이드, 디저트
- 좌측 또는 상단에 고정된 카테고리 탭 제공
- 선택한 카테고리가 시각적으로 강조되어야 한다.

### 4-3. 메뉴 목록
- 메뉴 이미지, 메뉴명, 가격 표시
- 메뉴 카드를 터치하면 상세 옵션 화면으로 이동
- 인기 메뉴, 신메뉴 뱃지 추가 가능

### 4-4. 메뉴 상세 및 옵션 선택
- 수량 선택
- 옵션 선택 예시
- 사이즈 선택
- 추가 토핑 선택
- 세트 변경 여부 선택
- 옵션 선택 후 장바구니 담기

### 4-5. 장바구니
- 선택 메뉴 목록 확인
- 수량 증가/감소
- 메뉴 삭제
- 총 주문 금액 표시
- 주문 계속하기 / 주문 진행하기 버튼 제공

### 4-6. 주문 확인
- 최종 주문 목록 표시
- 총 금액 확인
- 누락 또는 오주문 방지를 위한 최종 확인 단계 제공

### 4-7. 결제 안내 화면
- 실제 결제 모듈 없이도 결제 직전 안내 화면 구성 가능
- 예: 카드 결제를 진행해주세요 / 직원 호출 등

---

## 5. 화면 구성안

### 화면 1. 시작 화면
- 매장 로고
- 주문 시작 버튼
- 간단한 안내 문구

### 화면 2. 메뉴 선택 화면
- 카테고리 영역
- 메뉴 카드 리스트
- 우측 또는 하단 고정 장바구니 요약

### 화면 3. 메뉴 옵션 팝업 또는 상세 화면
- 메뉴 이미지
- 메뉴 설명
- 가격
- 옵션 선택 영역
- 장바구니 담기 버튼

### 화면 4. 장바구니 / 주문 확인 화면
- 선택한 메뉴 목록
- 수량 수정 버튼
- 총 금액
- 결제 진행 버튼

### 화면 5. 결제 안내 화면
- 결제 요청 메시지
- 주문 번호 또는 안내 문구

---

## 6. UI/UX 디자인 방향

### 6-1. 디자인 컨셉
- 전체적으로 깨끗하고 정돈된 화이트 베이스 디자인
- 불필요한 장식 없이 정보 전달 중심의 레이아웃
- 키오스크 특성상 멀리서도 잘 보이는 큰 타이포와 버튼 사용

### 6-2. 컬러 방향
- 메인 배경: 화이트
- 서브 배경: 아주 연한 그레이
- 포인트 컬러: 블랙 또는 딥 그레이, 필요 시 한 가지 브랜드 포인트 컬러만 사용
- 상태 컬러: 선택, 강조, 경고 상태가 명확히 구분되도록 제한적으로 사용

### 6-3. 타이포그래피
- 가독성이 높은 산세리프 계열 폰트 사용
- 제목, 가격, 버튼 텍스트는 크게 표시
- 설명 문구는 짧고 간결하게 유지

### 6-4. 버튼 및 인터랙션
- 터치 영역이 넓은 버튼 사용
- 주요 액션 버튼은 하단 또는 우측에 고정
- 누른 상태, 선택 상태가 명확히 보이도록 설계
- 애니메이션은 과하지 않고 빠르게 반응하도록 구성

### 6-5. 레이아웃 방향
- 정보 우선순위가 분명한 카드형 UI
- 카테고리, 메뉴, 장바구니의 영역 분리가 명확해야 함
- 사용자가 현재 어디에 있는지 알 수 있도록 단계 흐름 표시

## 7. 기술 스택 제안

### 7-1. 기본 구성
- HTML: 화면 구조 작성
- CSS: 화이트 베이스 UI 스타일링
- JavaScript: 동적 인터랙션 처리

### 7-2. JavaScript 역할
- 메뉴 데이터 렌더링
- 카테고리 필터링
- 옵션 선택 처리
- 장바구니 상태 관리
- 총 주문 금액 계산
- 화면 전환 또는 모달 제어

### 7-3. 데이터 구조 예시
```javascript
const menuItems = [
  {
    id: 1,
    category: "burger",
    name: "클래식 버거",
    price: 6500,
    image: "images/burger-01.jpg",
    popular: true,
    options: {
      size: ["single", "set"],
      extras: ["cheese", "bacon"]
    }
  }
];
```

## 8. 핵심 기능 상세 정의

### 8-1. 메뉴 출력 기능
- JavaScript로 메뉴 데이터를 반복 렌더링한다.
- 선택한 카테고리에 따라 목록이 필터링되어야 한다.

### 8-2. 장바구니 기능
- 메뉴 추가 시 장바구니에 항목이 누적된다.
- 동일 메뉴와 동일 옵션일 경우 수량을 증가시키는 방식도 고려할 수 있다.
- 총 금액은 항목 변경 시 즉시 재계산되어야 한다.

### 8-3. 옵션 선택 기능
- 필수 옵션과 선택 옵션을 구분한다.
- 필수 옵션이 선택되지 않으면 장바구니 추가가 불가능하도록 처리한다.

### 8-4. 주문 확인 기능
- 결제 직전 주문 내역 전체를 다시 확인할 수 있어야 한다.
- 삭제 또는 수량 변경이 가능해야 한다.

## 12. 한 줄 정리

이 프로젝트는 JavaScript를 기반으로 한 키오스크 주문 웹페이지를 제작하는 것으로, 화이트 베이스의 깔끔한 디자인과 직관적인 주문 흐름을 통해 빠르고 쉬운 사용자 경험을 제공하는 것을 목표로 한다.
