## 4. 핵심 작업 항목

### 4-1. 요구사항 정리
- 키오스크 주문 흐름을 단순하게 정의한다.
- 사용자 동선은 `시작 화면 → 메뉴 선택 → 옵션 선택 → 장바구니 → 주문 확인 → 결제 안내`로 정리한다.
- 이번 1차 구현 범위는 프론트엔드 화면과 주문 인터랙션까지로 제한한다.

### 4-2. 필수 기능 확정
- 시작 화면 표시
- 카테고리별 메뉴 목록 표시
- 메뉴 상세 및 옵션 선택
- 장바구니 추가, 삭제, 수량 변경
- 총 금액 계산
- 주문 확인 화면
- 결제 직전 안내 화면

### 4-3. 화면 목록 확정
- 화면 1: 시작 화면
- 화면 2: 메뉴 선택 화면
- 화면 3: 메뉴 상세/옵션 선택 팝업 또는 상세 영역
- 화면 4: 장바구니 및 주문 확인 화면
- 화면 5: 결제 안내 화면

### 4-4. 기본 폴더 구조 설계
- `index.html`
- `style.css`
- `script.js`
- `assets/`
- `assets/images/`

### 4-5. HTML 기본 구조 설계
- 헤더 영역: 매장명 또는 로고
- 카테고리 영역: 메뉴 분류 버튼
- 메뉴 리스트 영역: 카드형 메뉴 출력
- 장바구니 영역: 선택 메뉴와 총 금액 표시
- 모달 영역: 메뉴 상세 및 옵션 선택

---

## 5. 화면 구조 초안

## 시작 화면
- 로고 또는 브랜드명
- 주문 시작 버튼
- 간단한 안내 문구

## 메뉴 선택 화면
- 상단 헤더
- 카테고리 탭
- 메뉴 카드 리스트
- 우측 또는 하단 장바구니 요약

## 메뉴 상세 화면
- 메뉴 이미지
- 메뉴명
- 가격
- 옵션 선택
- 수량 선택
- 장바구니 담기 버튼

## 주문 확인 화면
- 선택한 메뉴 목록
- 수량 수정 버튼
- 삭제 버튼
- 총 주문 금액
- 결제 진행 버튼

## 결제 안내 화면
- 결제 요청 메시지
- 주문 확인 문구
- 처음으로 돌아가기 또는 완료 안내
